## 1. ОПИСАНИЕ ПРОЕКТА: «УЧЕТ ЗАЯВОК В РЕМОНТНОЙ МАСТЕРСКОЙ»
### 1.1.	Предметная область
Система предназначена для автоматизации учёта заявок на ремонт техники в мастерской. Она облегчает процесс приёма, отслеживания и выполнения заявок как для клиентов, так и для сотрудников. Взаимодействие с системой осуществляется через консольный интерфейс на Python.
### 1.2.	Цели системы
#### 1.2.1.	Для клиентов: 
Упростить процесс подачи заявок на ремонт и получение информации о статусе.
#### 1.2.2.	Для мастеров: 
Обеспечить удобный инструмент для обработки и ведения заявок.
#### 1.2.3.	Для администрации: 
Оптимизировать контроль за выполнением ремонтов и распределением задач.
#### 1.2.4.	Общая цель: 
Повысить эффективность работы ремонтной мастерской.
### 1.3.	 Пользователи системы
#### 1.3.1.	• Клиент:
Создание заявки на ремонт.
Просмотр статуса заявки.
Регистрация/вход.
#### 1.3.2.	 Мастер:
Просмотр назначенных заявок.
 	Обновление статуса (в работе, выполнено, отклонено).
#### 1.3.3.	Администратор:
Добавление и удаление мастеров.
Контроль выполнения заявок.
### 1.4.	Основные функции системы
#### Функционал для клиента:
1. Создание заявки: Подача новой заявки на ремонт выбранного устройства.
2. Мои заявки: Просмотр всех своих заявок с текущими статусами.
#### Функционал для мастера:
1. Мои заказы: Просмотр назначенных заказов и изменение их статуса.
#### Функционал для администратора:
1. Управление мастерами: Добавление, удаление и просмотр списка мастеров.
2. Контроль заявок: Просмотр всех заявок, распределение их между мастерами.
